System Features

FLIGHT
SYSTEMS

Every subsystem engineered for precision — from sensor fusion to motor mixing.

01 / IMU
IMU Sensor Fusion
MPU-6050 gyroscope and accelerometer fused via complementary filter. Real-time roll and pitch at 50 Hz. I2C address 0x68 on pins A4/A5.
02 / PID
PID Stabiliser
Dual-axis PID controller with configurable Kp, Ki, Kd. Anti-windup integrator clamping. Runs every 20 ms for 50 Hz control authority.
03 / ESC
Motor Mixing
X-frame quadcopter mixing on digital pins D9, D10, D11, D6. PWM signals 1000–2000 µs via Arduino Servo library. Rotor directions alternated for yaw authority.
04 / ALT
Altitude Sensing
HC-SR04 ultrasonic on D2 (trigger) and D3 (echo). Median-filtered distance readings in metres. Max range 4 m with ±3 mm accuracy.
05 / LCD
I2C LCD Display
LiquidCrystal_I2C 16×2 at address 0x27. Shows ARM state, altitude, roll, pitch, and throttle. Refreshes every 200 ms to prevent I2C bus saturation.
06 / SAFETY
ARM / DISARM
Push-button on D4 with 50 ms debounce. Hard motor lock when disarmed. PID integrators zero on arm. LED indicator on D13 mirrors arm state.
07 / THROTTLE
Throttle Control
10 kΩ potentiometer on A0 maps 0–5V to 1000–2000 µs PWM range. Deadband at low throttle prevents inadvertent arm. Capped during PID saturation.
08 / SERIAL
Serial Telemetry
Full telemetry stream at 9600 baud via USB. Roll, pitch, altitude, throttle, and all four motor values logged every control cycle for debugging and tuning.
09 / LOOP
Real-Time Loop
Deterministic 50 Hz control loop using micros()-based timing. Sensor read, PID compute, motor write — all sequenced within the 20 ms window at 16 MHz.
